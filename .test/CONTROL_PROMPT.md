# CONTROL_PROMPT — 对照组提示词

> 本文件包含给对照组的标准提示词。
> 在外部终端的 Claude Code 会话中，复制粘贴以下提示词。

---

请帮我创建一个基于浏览器的贪吃蛇小游戏。

**位置**：在 `test/test-normal/snake.html` 创建单个 HTML 文件。

**要求**：
1. 单 HTML 文件，所有 CSS 和 JS 内嵌
2. 方向键（↑↓←→）控制蛇移动方向
3. 蛇不能 180 度反向移动（如向右移动时不能立即向左）
4. 撞墙或撞自己身体时游戏结束
5. 吃食物得 10 分、蛇身增长一格
6. 显示当前分数（Score）和最高分（Best，localStorage 持久化）
7. 400x400 像素画布，20x20 像素一格
8. 蛇初始长度 3，初始位置在画布中央
9. 跨浏览器兼容（Chrome/Firefox/Safari）
10. 双击 HTML 文件即可在浏览器中打开运行
11. 不引用任何外部资源（CDN、npm 包等）
12. 游戏结束后显示 "Game Over" 并提供重新开始功能

请开始实现。完成后告诉我：
- 总共花了多少轮对话？
- 中间有没有遇到错误或需要返工的地方？
- 最终文件大小（行数）？
- 有没有"差点遗漏"或"后来才想起来加"的功能？

---

**测试说明**（仅给阅读本文件的用户看，对照组不需要知道这些）：

对照组将在独立 Claude Code 会话中执行以上提示。本目标（control-test）会在用户报告对照组完成后，对照 asdev 端和对照组的实现，量化对比：
- 交付质量（代码完整性、设计文档、验收报告）
- 成本与效率（对话轮数、token 消耗、时间）
- 可追溯性（决策记录、风险点）
- 出错率与返工率
