# 贪吃蛇游戏功能规格

> 对照测试任务规格：test/test-asdev/ 和 test/test-normal/ 两个目录都按此规格实现

## 任务目标

在指定目录下实现一个基于前端浏览器的贪吃蛇小游戏。

## 核心功能要求

### 1. 游戏控制
- 方向键（↑↓←→）控制蛇移动方向
- 不允许 180 度反向移动（如蛇向右移动时不能立即向左）
- 按空格键暂停/继续游戏
- 游戏结束后按 Enter 或点击按钮重新开始

### 2. 游戏规则
- 蛇初始长度为 3，初始位置在画布中央
- 蛇每次移动一格，移动速度固定（约 200ms/格）
- 吃到食物（红点）：蛇身增加 1 格，得分 +10
- 撞墙（超出画布边界）：游戏结束
- 撞到自己身体：游戏结束

### 3. UI 元素
- 分数显示（顶部或画布上方）："Score: X"
- 最高分记录（localStorage 持久化）："Best: Y"
- 游戏结束提示：显示 "Game Over! Final Score: X" 和重新开始按钮
- 画布大小：400x400 像素，使用 canvas 元素
- 网格：20x20 像素一格

### 4. 视觉要求
- 蛇身：绿色，头部稍深
- 食物：红色
- 背景：浅色（如白色或浅灰）
- 边界：可见的边框

## 技术约束

### 必须
- **单 HTML 文件**：所有 CSS 和 JS 内嵌在 `<style>` 和 `<script>` 标签中
- **零外部依赖**：不引用任何 CDN、npm 包或外部文件
- **跨浏览器兼容**：在 Chrome、Firefox、Safari 最新稳定版上都能正常运行
- **独立运行**：双击 HTML 文件即可在浏览器中打开运行

### 推荐
- 使用 `requestAnimationFrame` 或 `setInterval` 控制游戏循环
- 使用 `localStorage` 保存最高分
- 错误处理：用户输入无效时不崩溃

## 验收测试清单

实现完成后，应能通过以下测试：

- [ ] 打开 HTML 文件后立即看到游戏画面
- [ ] 按方向键蛇开始向对应方向移动
- [ ] 蛇不能 180 度反向（按相反方向键无效）
- [ ] 吃到食物后蛇身增长、分数增加
- [ ] 撞墙后游戏结束，显示 Game Over
- [ ] 撞到自己身体后游戏结束，显示 Game Over
- [ ] 按 Enter 或点击按钮可以重新开始
- [ ] 最高分记录在重新打开页面后仍然存在
- [ ] 分数显示正确
- [ ] 画布大小为 400x400，网格 20x20
